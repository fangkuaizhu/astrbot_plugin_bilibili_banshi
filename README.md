# B站搬石插件 (astrbot_plugin_bilibili_banshi)

随机从B站搜索视频并自动分享到群聊的 AstrBot 插件。
**群组独立版**：每群的开关、关键词、已发记录、推送模式均互不干扰。

## 功能

- **群组独立**：每群可独立开启/关闭搬石，独立管理关键词和历史
- **两种推送模式**：间隔推送（每N秒）或定时推送（指定时间点 HH:MM）
- **定时扫描**：短循环检测，每群按自身配置独立触发
- **自动记录新群**：首次收到群消息时自动绑定
- **去重**：每群独立记录最近 500 条已发送 BV，不重复发送
- **WebUI 设置页**：支持在 AstrBot WebUI 中对每群直接配置

## 安装

1. 将插件目录放到 AstrBot 的 `data/plugins/` 下
2. 重启 AstrBot（或在 WebUI 插件页手动加载）
3. 在目标群发送 `/banshi on` 开启

## 配置

### WebUI 设置（全局默认值）

安装后，在 AstrBot WebUI → 插件页 → 点击「B站搬石」卡片，可配置：
- **新群默认搜索关键词**：新群首次开启时使用的关键词列表
- **新群默认搜索页数**：新群首次开启时的搜索页数

### WebUI 设置页（每群独立）

进入插件 → `settings` 页面，可针对每群独立配置：
- 搬石开关
- 推送模式（间隔/定时）
- 搜索关键词
- 视频时长上限
- 搜索页数

### 指令配置（每群独立）

| 指令 | 说明 |
|------|------|
| `/banshi on` | 在本群开启定时搬石 |
| `/banshi off` | 在本群关闭定时搬石 |
| `/banshi now` | 本群立即搬一次 |
| `/banshi list` | 查看本群运行状态 |
| `/banshi kw` | 查看本群关键词列表 |
| `/banshi kw add <词>` | 为本群添加关键词 |
| `/banshi kw del <词>` | 从本群删除关键词 |
| `/banshi maxdur <秒>` | 设置本群视频最大时长（0=不限） |
| `/banshi mode interval` | 切换为间隔推送模式 |
| `/banshi mode scheduled` | 切换为定时推送模式 |
| `/banshi interval <秒>` | 设置本群间隔推送间隔 |
| `/banshi addtime <HH:MM>` | 增加一个定时推送时间点 |
| `/banshi deltime <HH:MM>` | 删除一个定时推送时间点 |
| `/banshi reset` | 清空本群已发送历史 |

所有指令都在 **发送指令的群** 内生效，不同群互不影响。

## 工作原理

1. 定时器每 30 秒检查一次
2. 遍历所有已开启搬石的群
3. 对每群独立判断是否满足触发条件：
   - **间隔模式**：距上次扫描是否超过设定间隔
   - **定时模式**：当前时间是否匹配定时点
4. 从该群的关键词列表中随机选一个 → 调用 Bilibili 搜索 API → 过滤已发送 → 随机选一个视频
5. 获取视频详情 → 构建文字卡片 → 发送到该群

**不下载视频文件，只发链接和文字信息**，无需 ffmpeg。

## 数据存储

- `data/plugin_data/astrbot_plugin_bilibili_banshi/config.json`：
  包含 `bound_groups`（已绑定群列表）和 `group_configs`（每群的独立配置）
- `data/config/astrbot_plugin_bilibili_banshi_config.json`：
  AstrBot WebUI 全局默认配置（自动同步）

## 注意事项

- Bilibili 搜索 API 无认证，有频率限制，短时间大量搜索可能被限流
- 建议每群分开时间段开启，或全局间隔不低于 30 秒
- 某群关闭后，该群的配置和历史保留，下次开启时沿用
